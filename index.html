1. Claude Codeとは？

Anthropic社が作った公式のCLIツール

CLI（Command Line Interface）とは？
マウスではなくキーボードで文字を打って操作するツール。
Macなら「ターミナル」、Windowsなら「コマンドプロンプト」のこと。

ターミナル上でClaudeと対話しながら
コーディングができる

何ができるの？

コード生成・編集

「ログイン機能を作って」と言えば
ファイルを直接作成・編集してくれる

コードベース理解

プロジェクト内の全ファイルを読んで
「この関数何してる？」に答えてくれる

コマンド実行

テスト実行、パッケージインストール、
git操作をAIが代わりにやってくれる

デバッグ

エラーログを見せれば原因を特定し
修正コードまで書いてくれる

従来のAIツールとの違い

ChatGPT / Gemini GitHub Copilot Claude Code
動作場所 ブラウザ エディタ内 ターミナル
ファイル編集 コピペが必要 補完のみ 直接書き換え
コマンド実行 不可 限定的 自由に実行
プロジェクト理解 貼った分だけ 開いたファイル 全体を横断
マルチファイル 手動コピペ 1ファイルずつ 複数同時に編集

ざっくりイメージ

ChatGPTが「アドバイスしてくれる先生」だとすると

Claude Codeは「一緒に手を動かしてくれるペアプログラマー」

You: このバグ直して
Claude: 原因はここです。修正しますか？
→ [Allow] を押すとファイルが書き換わる
Claude: テスト実行して確認しますか？
→ [Allow] を押すとテストが走る → 結果表示

2. 先に覚えておく用語

Claude Codeで出てくる言葉を整理

基本用語

ターミナル / シェル
文字を打ってPCを操作する画面。
・Mac → 「ターミナル.app」（Spotlight で "ターミナル" と検索）
・Windows → 「PowerShell」（スタートメニューで検索）
cd（フォルダ移動）などのコマンドを打つ場所。
CLI（Command Line Interface）
GUIの反対。マウスではなくキーボードだけで操作するソフト。
Claude Codeはこの形式。
npm（Node Package Manager）
JavaScriptのパッケージ管理ツール。npm install でソフトをインストールできる。
Claude Codeのインストールにも使える（レガシー方式）。

Claude Code固有の用語

コンテキストウィンドウ
Claudeが「今この瞬間に覚えていられる情報量」の上限。
会話が長くなるほど埋まっていき、古い情報から忘れてしまう。
人間の「作業机の広さ」のようなもの。
セッション
1回の会話のまとまり。claude を起動してから終了するまでが1セッション。
セッションを跨ぐと会話内容はリセットされる（メモリ機能で一部引き継ぎ可能）。

Claude Code固有の用語（続き）

トークン
AIが文章を処理する最小単位。日本語1文字 ≒ 1〜3トークン。
API利用時はトークン数で料金が決まる。/cost で確認可能。
パーミッション（Permission）
Claudeに与える「許可」のこと。ファイルを編集していいか、
コマンドを実行していいか、を毎回 or 自動で判断する仕組み。
MCP（Model Context Protocol）
Claudeと外部サービス（GitHub, Slack等）を繋ぐ共通規格。
「USBのような統一規格」と思えばOK。

Claude Code固有の用語（続き）

サブエージェント
Claudeが内部で起動する「分身」。大きなタスクを並列処理したり、
調査専門のエージェントに情報収集させたりする。
Hooks（フック）
「特定のタイミングで自動実行するスクリプト」。
例：ファイル保存後に自動でコード整形を走らせる、など。
釣り針（Hook）のように、イベントに引っ掛けて処理を実行する。
Skills（スキル）
Claude Codeに追加できる「カスタム機能」。
Markdownファイルに手順を書いておくと、スラッシュコマンドで呼び出せる。

3. インストールと初期設定

必要なもの

  • Anthropicアカウント（claude.aiで作成）
  • いずれかのプラン（無料でも使える！）
プラン料金Claude Code特徴
Free無料使える回数制限あり。お試しに
Pro月額 $20使える制限緩和。普段使いに十分
Max 5x月額 $100使えるたっぷり使える。開発者向け
Max 20x月額 $200使えるほぼ無制限。ヘビーユーザー向け
API Key従量課金使える使った分だけ。組織・大量利用向け

インストール手順（Mac / Linux）

# 方法1: ネイティブインストール（推奨）
curl -fsSL https://claude.ai/install.sh | bash

# 方法2: Homebrew（macOSのみ）
brew install --cask claude-code

# 方法3: npm（Node.js必要）
npm install -g @anthropic-ai/claude-code
curl とは？
URLからデータをダウンロードするコマンド。
curl ... | bash = 「スクリプトをダウンロードして即実行」

インストール手順（Windows）

# 方法1: PowerShell（推奨）
irm https://claude.ai/install.ps1 | iex

# 方法2: コマンドプロンプト
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

# 方法3: WinGet
winget install Anthropic.ClaudeCode

# 方法4: npm（Node.js必要）
npm install -g @anthropic-ai/claude-code
PowerShellとは？
Windows標準の高機能ターミナル。スタートメニューで「PowerShell」と検索して起動。
irm = Invoke-RestMethod（ダウンロード）、iex = Invoke-Expression（実行）

インストール確認

# バージョン確認
claude --version

# 問題がないか診断
claude doctor
うまくいかない時：
・command not found → ターミナルを再起動
・Node.jsが無い → Mac: brew install node / Win: winget install OpenJS.NodeJS
・Windows: 「管理者として実行」でPowerShellを開く

初回起動と認証

# どのフォルダでもOK
claude

初回はブラウザが自動で開く

Free / Pro / Maxの場合

ブラウザでAnthropicアカウントにログイン
→ 自動で認証完了
無料プランでもこれでOK！

API Keyの場合

export ANTHROPIC_API_KEY=sk-...
claude
export とは？
ターミナルに「環境変数」をセットするコマンド。
API Keyをセットすると、claudeがそのキーで認証する。

4. 基本の使い方

まずは触ってみよう

起動のしかた

# プロジェクトフォルダに移動してから起動
# Mac / Linux
cd ~/Desktop/my-project

# Windows
cd C:\Users\ユーザー名\Desktop\my-project

claude
重要： 起動したフォルダ = Claudeの作業範囲
必ずプロジェクトのルートフォルダで起動すること！
デスクトップで起動すると、デスクトップ全体が操作対象になってしまう。

対話してみよう

You: Pythonで簡単なTodoアプリを作って

Claude: main.pyを作成します。
📝 Write main.py
→ [Allow] を押すとファイルが作られる

You: 動かしてみて

Claude: python3 main.py を実行します。
🔧 Bash: python3 main.py
→ [Allow] を押すとコマンドが実行される

Claudeが使う「ツール」

Claudeは内部で色々なツールを使い分けている

ツール名何をするか例
Readファイルを読むmain.pyの中身を確認
Write新規ファイル作成新しいファイルを作る
Edit既存ファイル編集3行目を書き換える
Bashコマンド実行pip install, git commit
Globファイル検索*.pyのファイル一覧
Grepコード内検索"def login" を含むファイル

ツールの使用時に許可を求められる（パーミッション設定で制御可能）

権限の確認画面

Claudeが何かする前に必ず確認が出る

Claude wants to: Edit main.py

  [y] Allow    [n] Deny    [a] Allow Always
  • y（Allow） — 今回だけ許可
  • n（Deny） — 拒否（別の方法を提案してくれる）
  • a（Allow Always） — 以降この操作は自動許可
最初は全部 y で進めて感覚をつかもう！
慣れてきたら a で効率化。

セッションの管理

会話の継続・再開ができる

対話中に使う（スラッシュコマンド）

# セッションに名前を付ける
/rename todo-app-dev

# 過去のセッション一覧から再開
/resume

起動時に使う（ターミナルで打つ）

# 前回の会話を継続して起動
claude -c

# 過去のセッションを選んで起動
claude -r
いつ新しいセッションにする？
・タスクが完了した時
・全然違う作業に移る時
・コンテキストが足りなくなった時（後述）
上級者向け：ワンショットモード
claude -p "質問" で対話セッションを開かず1回だけ実行できる。
スクリプトに組み込んで自動化する時に便利（最初は覚えなくてOK）。

5. スラッシュコマンド

対話中に / で始まる特殊コマンドが使える

Claudeへの指示ではなく、Claude Code自体への命令

最もよく使う4つ

/help

使い方のヘルプを表示。
困ったらまずこれ。

/clear

会話をリセットして最初から。
コンテキストも全解放。

/compact

会話を要約して圧縮。
コンテキストが足りない時に。

/cost

今のセッションで使った
トークン数と料金を表示。

プロジェクト管理系

/init

CLAUDE.mdを自動生成。
プロジェクト開始時に実行。

/permissions

今の権限設定を確認・変更。
何が許可されてるか一覧。

/model

AIモデルを切り替え。
Opus（高性能）↔ Sonnet（高速）

/memory

メモリの中身を確認・編集。
Claudeが何を記憶してるか。

セッション管理系

/resume

過去のセッション一覧。
中断した作業を再開。

/rename [名前]

セッションに名前を付ける。
後で探しやすくなる。

/context

コンテキスト使用量の詳細。
何がどのくらい使ってるか。

/config

設定を対話的に変更。
テーマやモデルの設定など。

Tips: claude -c でも前回の会話を継続できる（起動時オプション）

/compact を深掘り

コンテキスト管理で最も重要なコマンド

# 基本：全体を圧縮
/compact

# フォーカスを指定して圧縮
/compact 最近のAPI変更に焦点を当てて

# 特定の情報を保持
/compact テスト結果は残して
コンテキストが足りなくなると：
・Claudeがさっき話したことを忘れる
・同じことを何度も聞いてくる
・画面下のバーが赤くなる
→ /compact で圧縮するか /clear でリセット

6. CLAUDE.md — 実践編

プロジェクトの取扱説明書をClaudeに渡す

CLAUDE.mdとは？

プロジェクトのルート（一番上のフォルダ）に置く
Markdownファイル

書いておくこと

  • 技術スタック
  • ビルド・テストのコマンド
  • コーディングルール
  • やってはいけないこと
  • ディレクトリ構成

メリット

  • 毎回説明しなくていい
  • セッションが変わっても有効
  • チームで共有できる
  • Claudeの精度が上がる

一番簡単な作り方

# プロジェクトフォルダで
/init

Claudeがプロジェクトを自動分析して
CLAUDE.mdを生成してくれる！

手動で作る場合：
プロジェクトフォルダに CLAUDE.md を作成するだけ。
特別なフォーマットは不要、普通のMarkdownでOK。

実践例：Webアプリの場合

# 飲食店予約アプリ

## 技術スタック
- バックエンド: FastAPI (Python 3.11)
- フロントエンド: HTML + Vanilla JS（フレームワーク不使用）
- DB: SQLite
- 公開: Cloudflare Tunnel

## 開発コマンド
- サーバー起動: `uvicorn main:app --reload --port 8000`
- テスト: `pytest tests/ -v`
- 依存追加: `pip install パッケージ名 && pip freeze > requirements.txt`

## コーディングルール
- Pythonの変数名・関数名はスネークケース（例: get_user_name）
- HTMLテンプレートは templates/ に配置
- APIレスポンスは必ず JSON で返す
- 日本語コメント OK

## 絶対やらないこと
- .env ファイルに直接APIキーを書かない
- git push を勝手にしない（確認してから）
- 既存のデータベースを DROP しない

実践例：データ分析の場合

# 売上データ分析プロジェクト

## 概要
月次の売上CSVを分析してレポートを生成する

## 環境
- Python 3.11 + pandas + matplotlib
- Jupyter Notebook は使わない（.pyファイルのみ）
- グラフは static/graphs/ に PNG で保存

## データについて
- data/ フォルダにCSVが入っている
- 文字コードは Shift-JIS（cp932）
- 日付列は "YYYY/MM/DD" 形式
- 金額にカンマが入っている場合あり（"1,234"）

## お願い
- グラフのフォントは日本語対応にする
- 分析結果は results/ に出力
- 元データは絶対に書き換えないこと

CLAUDE.mdの配置場所

プロジェクト用

my-project/CLAUDE.md

そのプロジェクトだけに効く
gitでチーム共有可能

個人グローバル

~/.claude/CLAUDE.md

全プロジェクト共通
個人の好みを書く

ルール分割

.claude/rules/*.md

大きいプロジェクト向け
ルールをファイルで整理

グローバルCLAUDE.mdの例：
「日本語で回答して」「コミットメッセージは日本語で」「説明は簡潔に」
→ 全プロジェクトで毎回言わなくて済む！

CLAUDE.md 書き方のコツ

良い書き方

  • 具体的なコマンドを書く
  • 「やらないこと」を明記
  • ディレクトリ構成を書く
  • 200行以内に収める

避けた方がいい書き方

  • 「いい感じに」等の曖昧指示
  • 長すぎる（コンテキストを圧迫）
  • 矛盾する指示
  • 古い情報の放置
注意： CLAUDE.mdはセッション開始時に毎回読み込まれます。
長すぎるとコンテキストを食ってしまう → 簡潔に！

# SKILL.md を作成
# → 中身は次のスライド
スキルの保存場所：
・.claude/skills/スキル名/SKILL.md → そのプロジェクト用

カスタムスキルを作ってみよう（2）

例：コードレビュースキル

---
name: review
description: コードレビューを実行する
---

# コードレビュー手順

以下の観点でレビューしてください：

## チェック項目
1. **セキュリティ**: SQLインジェクション、XSSの脆弱性はないか
2. **エラー処理**: 例外が適切にハンドリングされているか
3. **可読性**: 変数名・関数名は分かりやすいか
4. **パフォーマンス**: N+1クエリなど非効率な処理はないか

## 出力形式
- 問題の深刻度: 高 / 中 / 低
- 該当箇所: ファイル名と行番号
- 改善案: 具体的なコード例を添えて

カスタムスキルを作ってみよう（3）

使ってみる

You: /review

Claude: コードレビューを実行します。

📖 Read main.py
📖 Read utils.py
📖 Read models.py

Claude:
🔴 高 main.py:42 - SQLインジェクションの脆弱性
  f"SELECT * FROM users WHERE id={user_id}"
  → パラメータバインディングを使用してください

🟡 中 utils.py:15 - 例外が握りつぶされている
  except: pass → 適切なエラー処理を追加
...

実践的なスキル例

/deploy

テスト→ビルド→デプロイ
の手順を自動実行

/fix-issue [番号]

GitHub Issueを読んで
修正を自動実装

/daily-report

今日のgit commitから
日報を自動生成

/add-api [名前]

API エンドポイントの
テンプレートを自動生成

引数付きスキルの例

引数（ひきすう）とは？
コマンドに渡す追加情報のこと。
例：/add-api ユーザー登録 の「ユーザー登録」が引数。
スキル内で $ARGUMENTS と書くと、渡された引数に置き換わる。 
---
name: add-api
description: 新しいAPIエンドポイントを追加する
---

# APIエンドポイント追加

「$ARGUMENTS」という名前のAPIを作成してください。

## 作成するもの
1. `routes/` にルートファイル
2. `models/` にリクエスト/レスポンスモデル
3. `tests/` にテストファイル
# こう打つと…
/add-api ユーザー登録

# ↓ Claudeにはこう伝わる
「ユーザー登録」という名前のAPIを作成してください。

動的コンテキスト注入

スキル内でコマンドの実行結果を自動で埋め込める仕組み

どういうこと？
普通のスキルは「固定の文章」だけど、!`コマンド` と書くと
スキルが呼ばれた瞬間にそのコマンドが実行され、
結果がスキルの文章に埋め込まれる。つまり毎回最新の情報が使われる。

SKILL.mdに書く内容

最近の変更：
!`git log --oneline -5`

上記の変更をレビューして

Claudeに実際に届く内容

最近の変更：
a1b2c3d ログイン機能追加
e4f5g6h DB接続修正
i7j8k9l READMEの更新

上記の変更をレビューして

!`コマンド` の部分が、実行結果に置き換わるイメージ

スキルの高度な設定

SKILL.mdの先頭にある --- で囲まれた部分で
スキルの動作を細かく制御できる

--- で囲まれた部分とは？
「フロントマター」と呼ばれる設定エリア。
name と description は必須。それ以外はオプション（任意）。 
---
name: deploy
description: 本番デプロイを安全に実行
disable-model-invocation: true
allowed-tools: Bash, Read
---
設定名意味使い道
name/コマンド名になる/deploy で呼び出せる
descriptionこのスキルの説明Claudeが自動で使うか判断する材料
disable-model-invocation自動起動を禁止デプロイ等、勝手に動くと困るもの
allowed-tools使えるツールを制限安全のため読み込みだけにする等

最初は name と description だけ書けば十分！

8. パーミッション（権限管理）

Claudeに何を許可するかを制御する

なぜ権限管理が必要？

Claudeは強力だからこそ危険もある：
・重要なファイルを間違って削除
・本番サーバーにデプロイ
・APIキーの入ったファイルをgit push

→ だから「何をしていいか」を事前に決めておく

3つのモード

Ask（デフォルト）

毎回確認

ファイル編集・コマンド実行の
たびに許可を求められる

{

    "allow": [
      "Read",

      "Bash(pytest *)",
      "Bash(git status)",
      "Bash(git diff *)",

    "deny": [

      "Bash(git push *)",
      "Bash(DROP TABLE *)"
    ]
  }
}

MCPサーバー

Model Context Protocol — 外部サービスとの接続

MCPのイメージ：
USBポートのような「共通の差し込み口」。
GitHub、Slack、データベースなど色々なサービスを
同じ仕組みでClaudeに接続できる。

GitHub MCP

Issue読み取り、PR作成、
コードレビュー

Slack MCP

チャンネル読み取り、
メッセージ送信

DB MCP

SQLクエリ実行、
テーブル構造確認

Sentry MCP

エラー監視、
バグ分析

MCPサーバーの追加方法

# HTTPサーバー（クラウドサービス）
claude mcp add --transport http sentry https://mcp.sentry.dev/mcp

# ローカルサーバー（自分のPCで動かす）
claude mcp add --transport stdio github \
  -- npx @anthropic-ai/mcp-github

# 確認
claude mcp list

# 削除
claude mcp remove github

初心者のうちは無理に使わなくてOK。必要になった時に追加。

Hooks（フック）

「○○した時に△△を自動実行」という仕組み

フックのイメージ：
「ドアが開いたらチャイムが鳴る」のような自動トリガー。
Claudeの操作に「引っ掛けて（Hook）」スクリプトを自動実行する。
イベントタイミング使用例
PreToolUseツール実行前保護ファイルへの書き込みをブロック
PostToolUseツール実行後ファイル保存後にフォーマット
Notification通知時デスクトップ通知を送信
Stop応答完了時自動テスト実行

Hooks設定例

ファイル保存後に自動でコード整形する例

// .claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "npx prettier --write \"$FILE_PATH\""
          }
        ]
      }
    ]
  }
}
初心者向け：
Hooksは上級者向け機能です。最初は気にしなくてOK。
「こんなことができるんだ」程度に覚えておいてください。

コンテキスト管理

Claude Codeを使いこなす最も重要な概念

コンテキストウィンドウとは

Claudeの「作業机」の広さ

机の上に資料を広げて作業するイメージ。
・会話の内容
・読んだファイル
・実行したコマンドの出力
・CLAUDE.mdの内容
全てが机に載っている。机がいっぱいになると古い資料が落ちる。
コンテキストが溢れると：
・さっき話したことを忘れる
・同じ質問を繰り返す
・「自動圧縮」が走って古い情報が要約される
→ 画面下のバーが赤くなったら要注意！

コンテキスト管理のコツ

  1. 1タスク1セッション — タスクが終わったら /clear で新規
  2. /compact を活用 — 長い会話は途中で圧縮
  3. CLAUDE.md を簡潔に — 長すぎると常にコンテキストを食う
  4. 大量のログを貼らない — 必要な部分だけ見せる
  5. セッション名を付ける — 後で再開する時に探しやすい

10. 実践デモ

実際にアプリを作ってみよう！

デモの流れ

  1. プロジェクトフォルダを作成
  2. claude を起動
  3. /init でCLAUDE.mdを生成
  4. 「○○アプリを作って」と指示
  5. 生成されたコードを確認・許可
  6. 「テスト書いて実行して」と追加指示
  7. エラーがあれば「直して」で修正
  8. /commit でgitに保存

※ ライブデモで実演します

実際のやり取り例

You: FastAPIで体重記録アプリを作って。
     体重と日付を登録・一覧表示できるようにして。
     SQLiteに保存して。

Claude: 以下のファイルを作成します：
  1. main.py - FastAPIアプリ本体
  2. models.py - データベースモデル
  3. templates/index.html - フロントエンド
  4. requirements.txt - 依存パッケージ

📝 Write main.py [Allow]
📝 Write models.py [Allow]
📝 Write templates/index.html [Allow]
📝 Write requirements.txt [Allow]

Claude: パッケージをインストールして起動しますか？
🔧 Bash: pip install -r requirements.txt [Allow]
🔧 Bash: uvicorn main:app --reload [Allow]

効果的な指示のコツ

悪い指示の例

いい感じのアプリ作って
全部一気に完成させて
前に言ったやつ修正して
（コンテキストから消えてる場合）

良い指示の例

FastAPIで体重記録アプリを作って。
SQLiteに保存して。
まずDBモデルを作って。
次にAPIエンドポイント。
#main.py の35行目の
バリデーションを修正して

指示のコツまとめ

  • 具体的に — 技術名、ファイル名を指定
  • 段階的に — 一度に全部ではなく少しずつ
  • 確認しながら — Allow/Denyで制御を保つ
  • ファイルを指定 — #main.py で明示
  • 修正も自然言語 — 「ここをこう変えて」でOK
  • 困ったらEscape — いつでも中断できる安心感

困ったときは

状況対処法
暴走してるEscape で即停止
変なことしてるn（Deny） で拒否
会話がおかしい/clear でやり直し
コンテキスト不足/compact で圧縮
毎回同じこと言ってるCLAUDE.md に書く
特定作業を自動化したいSkills を作る
そもそも使い方わからん/help

まとめ

機能一言で最初に覚える？
基本の対話自然言語でコーディング必須
スラッシュコマンド/clear, /compact 等必須
CLAUDE.mdプロジェクトのルール書必須
パーミッション何を許可するか必須
コンテキスト管理記憶の限界を意識重要
Skillsカスタムコマンド便利
メモリセッション間の記憶自動で動く
MCP外部サービス接続必要時に
Hooks自動実行トリガー上級者向け
サブエージェント並列処理の分身自動で動く

学習の順序

  1. Step 1: インストール → claude 起動 → 何か作ってみる
  2. Step 2: /init で CLAUDE.md を作る
  3. Step 3: /compact, /clear でコンテキスト管理
  4. Step 4: パーミッション設定を調整
  5. Step 5: カスタムスキルを作ってみる
  6. Step 6: 必要に応じてMCP, Hooksを導入
大事なこと： 全部覚えなくていい！
まずはStep 1〜3だけで十分実用的に使えます。
